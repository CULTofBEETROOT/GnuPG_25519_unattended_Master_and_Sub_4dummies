# Генератор головних та підключів без нагляду GnuPG 25519
# & Форматер ключів USB-Luks (gpg-key storage) для ключів.

Огляд
Цей репозиторій надає набір скриптів для генерації головного та підключів GPG за допомогою USB-ключа з шифруванням LUKS. Скрипти розроблені для користувачів без root-доступу, але для певних операцій потрібен root-доступ. Ця конфігурація адаптована для Debian Trixie та призначена для користувачів, які можуть працювати на старому обладнанні без доступу до Інтернету під час процесу.

Важливі примітки
* Порядок виконання: Скрипти необхідно запускати у вказаному порядку: ПЕРШИЙ, ДРУГИЙ та ТРЕТІЙ. Не пропускайте жодних кроків, навіть якщо крок завершився невдачею.
* Середовище: Переконайтеся, що у вас є необхідні файли:

FIRST.sh

SECOND.sh

THIRD.sh

(за бажанням: batchproduction_ofTHIRD.sh)

...у каталозі завантажень перед початком.

# ДЛЯ ПАКЕТНОГО ВИРОБНИЦТВА (краще!)

* Якщо у вас є кілька адрес електронної пошти, для яких ви хочете згенерувати ключі GPG, запустіть їх від імені користувача root або за допомогою sudo, виконавши такі команди:

```bash
sudo bash /home/$USER/Downloads/FIRST.sh;
sudo bash /home/$USER/Downloads/SECOND.sh;
```

* Створіть два файли:

один зі стовпцем власників,

другий зі стовпцем адрес електронної пошти

та введіть у них поля аргументів у шаблоні:

```bash
/home/$USER/Downloads/batchproduction_ofTHIRD.sh <path/to/and/ownerfile.txt> <path/to/and/addressfile.txt>;
```

Цей останній скрипт дозволяє вводити списки адрес електронної пошти та їхніх відповідних власників, створюючи окремі bash-скрипти для кожної адреси електронної пошти.

* Коли ви запускаєте згенеровані окремі скрипти, вони створюють ключі GPG, що зберігаються на вашому USB-ключі, зашифрованому за допомогою LUKS. Важливо: OpenGPG рекомендує негайно використовувати, а потім видаляти будь-які самостійно незашифровані закриті ключі, щоб уникнути зберігання конфіденційної інформації.

# АБО ВИБЕРІТЬ ВИРОБНИЦТВО ОДНОГО КЛЮЧА (інтуїтивно зрозуміло).

Кроки, які потрібно виконати

Крок 1: Запустіть перший скрипт
1. Відкрийте термінал.
2. Виконайте наступну команду від імені користувача root або за допомогою sudo:

```bash
 bash /home/$USER/Downloads/FIRST.sh
```

Крок 2: Створення USB-ключа за допомогою LUKS
1. Почніть цей крок у режимі root:

```bash
 bash /home/$USER/Downloads/SECOND.sh
```

Крок 3: Збереження GPG Master та підрозділів на USB-пристрої
1. Почніть цей крок у режимі root. Можливо, вам знадобиться змінити дозволи скрипта, щоб зробити його виконуваним.
Використайте наступну команду:

```bash
 chmod +x /home/$USER/Downloads/THIRD.sh
```

2. Потім запустіть скрипт:

```bash
 /home/$USER/Downloads/THIRD.sh
```

Заключний крок: Безпечне відключення USB-пристрою

Після успішного виконання всіх кроків переконайтеся, що ви безпечно відключили свій USB-пристрій LUKS, щоб запобігти потенційним проблемам під час його повторного підключення:

```bash
sudo umount "/mnt/usb_gpg"
sudo cryptsetup luksClose myusb_key
```

або наступного разу, коли ви підключите свій пристрій, у вас можуть виникнути труднощі.

Висновок

Виконуючи ці кроки, ви можете безпечно створювати та зберігати свій GPG Master та підрозділи на USB-ключі, зашифрованому за допомогою LUKS. Завжди пам’ятайте про обережне поводження зі своїми закритими ключами та дотримуйтесь найкращих практик безпеки. Якщо у вас виникнуть запитання або проблеми, зверніться до документації або зверніться за допомогою до спільноти.
